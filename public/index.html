🎓 UNION Attack Tutorial — Follow these steps exactly as an attacker would

A UNION attack allows an attacker to extract data from other tables by appending a second query to the original one.

STEP 1 — Confirm Injection Point

Before extracting data, an attacker first confirms the field is injectable. They do this by entering a single quote ' and observing if the application throws an error.

Like checking if a door is unlocked before trying to open it — you don't kick it down immediately.

STEP 2 — Count the Columns

UNION SELECT requires the same number of columns as the original query. Attackers try ORDER BY 1, ORDER BY 2... or UNION SELECT 1, UNION SELECT 1,2... until an error disappears, revealing the column count.

STEP 3 — Extract the Users Table

Now the attacker knows there are 6 columns. They replace the dummy numbers with actual column names from the users table.

STEP 4 — Extract Secret Configuration

An attacker doesn't stop at users. They probe for other tables containing API keys, secrets, and credentials.

🎓 What you just did — In 4 steps you:

  • Found an injection point
  • Determined the query structure
  • Extracted all user credentials
  • Stolen production secrets

This is exactly what automated tools like sqlmap do in seconds.

🛡️ ONE LINE fixes all of this:
Use prepared statements / parameterized queries.
